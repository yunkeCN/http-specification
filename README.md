# 云客http接口规范

## 初阶

### 字段命名规范

使用下划线格式

### 数据类型保持一致

例如：规定是数字, 不要返回字符串的数字

### 字段的状态需标示全面且清晰

例如, is\_checked, 0：选中, 1：为选中

### 接口输出的颗粒度

考虑页面性能, 尽可能合并

### 返回数据通用结构定义
需包含一下结构:
- `retCode`, 业务状态, 异常时表示业务错误码
- `msg`, 提示信息, 异常时表示异常信息
- `data`, 数据信息

### 不要包含无用字段

### 返回完全JSON格式

属性值不可为json字符串, 例如: data属性值不对

```js
{"data": "{\"key\":1}"}
```

### 提交数据长度超过1024b的要采用post方式

## 进阶

### 一个 key 对应的数据, 不超出： 数值、字符串、单层数组、单层 plain object
